开篇介绍

微软BI SSRS 2012 Metro UI 高端报表视频教程 （http://www.hellobi.com/course/15）课程从2014年6月开始准备，于2014年9月在 天善BI学院 （http://www.hellobi.com）正式上线。

本课程共包含 55个视频，35个原创基础案例，2个原创设计提高案例，其它扩展若干。1056余分钟，共计17小时。

学习方式：每一个视频案例均为现场编码，所有技术实现的细节都交代的很清楚，因此可以跟着视频边看边练习。

目标检测：每练习完一个视频案例，在不看视频的前提下如果可以独立的、熟练的实现本案例，则说明已经掌握了案例中所涉及到的知识技能点；反之，则说明还未完全掌握此案例，也未达到一个熟练的 SSRS 报表开发人员应该要具备的开发能力。

这门课程也是和 ETL 课程一样进行的体系性讲解，35个基本案例让大家足以掌握 SSRS 报表开发过程中常用的开发技能。并且在高级设计篇提高大家的报表设计能力，特别用到了 WIN 8 风格 METRO UI设计理念。这是我在微软的SSRS项目上所用到的一些设计方法，深受客户喜欢。并且在我目前的很多项目中，我大量的使用到了这些理念，所制作的报表在风格上有非常高的辨识度。所有案例均为原创，不按照书本千篇一律的讲解和 PPT 的讲解模式。每一个案例从无到有，也是一步一步的拖，拉，拽，写代码实现。每一个案例大家都可以做的出来。

WIN 8 系统在微软内部刚推出的时候，一下子就喜爱上了这种扁平化的设计风格，于是就把这种风格搬进了我的项目中。

无参数化的设计给客户很大的惊喜！这也是我做报表的一贯风格，能不用参数就尽量不用参数（用户无需输入任何参数，所有参数后台控制）。

用户的选择，钻取直接是在页面上点击完成的。不需要选择 City 参数，或者输入 City 一样可以实现界面的切换效果。对比之前印度的设计方的报表，新一版的报表 DEMO 原型让微软客户非常满意。

以下是 Google, Bing, Baidu 中对 SSRS Metro UI 关键字的搜索效果。（展示并不是为了炫耀，而只是想说明我们课程的原创性，我们的课程制作是非常用心的，每一个案例的制作都是原创的。同时也希望每一个 BI 开发者可以认识到 - 虽然 SSRS 报表很多人认为不好看，但是我们也依然可以做得非常漂亮）。

 

培训课程案例分类介绍

SSRS报表从入门到高级技能案例

35以上个不同的技能案例，包括案例查询代码以及全部界面设计均为 BIWORK 原创。总结了 BIWORK 在微软 BI 开发领域90%以上反复出现和常用的技能与案例，每一个案例均穿插着不同的知识点。灵活掌握和使用这些知识技能点，在开发技能储备上将足够应付在SSRS 报表开发过程中大部分报表开发需求。

SSRS 报表设计提高篇

尽管微软 BI 报表开发工具在前端展现上不尽如人意，但是作为一名报表开发人员应该如何突破这些局限，在SSRS 报表的构思与设计上变通，通过一些处理技巧让报表变得美观且更实用。一个好的报表不仅仅要学会展现数据，并且要展现的适当，给人耳目一新的感觉，增强用户体验让用户清晰直观的找到他们所需要的信息。BIWORK 将分享他的报表构思与设计经验，让你的 SSRS 报表生动起来。

BI 报表开发知识扩展篇

  • 介绍Office Excel 中PowerPivot 的使用 
  • 介绍报表的用户分类：Business User, Daily Operator, Manager 等，不同的用户有不同的报表需求。 
  • 介绍报表的分类：Ad-hoc Report, Dashboard, Transaction Static Report, Multi-Dimension Report 等。 
  • 介绍目前比较流行的 Tableau 和 QlikView 前端报表开发工具，适用范围，与微软 BI 报表开发异同等。
  • 注：所有案例均基于 SQL Server 2012 关系型数据库讲解，不适用于 SSAS 分析服务数据库和 MDX 查询。但可以通过这个课程的学习在 SSRS 报表前端开发阶段提高报表设计和解决问题的能力。
